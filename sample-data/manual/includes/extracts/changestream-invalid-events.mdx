  You cannot use `resumeAfter` to resume a change stream after an
  <Ref url="change-event-invalidate">invalidate event</Ref> (for example, a collection
  drop or rename) closes the stream. Instead, you can use
  <Ref url="change-stream-start-after">startAfter</Ref> to start a new change
  stream after an <Ref url="change-event-invalidate">invalidate event</Ref>.
