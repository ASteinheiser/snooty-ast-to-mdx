# Query Documents

<DefaultDomain>
  mongodb
</DefaultDomain>

<Contents local={true} backlinks="none" depth={{"$numberInt":"1"}}>

<DismissibleSkillsCard skill="Query Optimization" url="https://learn.mongodb.com/skills?openTab=query" />

To query documents, specify a <Ref url="query predicate">query predicate</Ref> indicating the
documents you want to return. If you specify an empty query predicate
(`{ }`), the query returns all documents in the collection.

You can query documents in MongoDB by using the following
methods:

<Include source="/includes/fact-methods.rst">
  * Your programming language's driver.

  * The [MongoDB Atlas UI](https://www.mongodb.com/docs/atlas/). To learn more, see
    <SubstitutionReference name="atlas-ref"><Ref url="query-documents-atlas-ui">Query Documents with MongoDB Atlas</Ref></SubstitutionReference>.

  * [MongoDB Compass](https://www.mongodb.com/docs/compass/current/).
</Include>

For reads to <Ref url="replication">Replica sets</Ref> and replica set

### Query Result Format

When you run a find operation with a MongoDB driver or `mongosh`, the
command returns a <Ref url="cursors">cursor</Ref> that manages query results. The query
results are not returned as an array of documents.

To learn how to iterate through documents in a cursor, refer to your
[driver's documentation](https://www.mongodb.com/docs/drivers/). If you are using `mongosh`, see
<Ref url="read-operations-cursors">Iterate a Cursor in `mongosh`</Ref>.
